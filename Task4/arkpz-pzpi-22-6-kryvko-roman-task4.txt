4 РОЗРОБКА IoT КЛІЄНТА (БІЗНЕС-ЛОГІКИ ТА ФУНКЦІЙ НАЛАШТУВАННЯ)
4.1 Мета роботи
На лабораторній роботі №4 потрібно розробити програмне забезпечення для IoT або SmartDevice пристрою, створеного на базі будь-якої поширеної на сьогодні платформи, придатної для реалізації вбудованих систем (Embedded System).


4.2 Опис інженерних рішень
IoT-клієнт розроблений для збору та передачі даних із сенсорів у теплиці. Система дозволяє отримувати показники навколишнього середовища та надсилати їх на сервер для подальшого аналізу. Використані технології: мова програмування C++, бібліотеки cpprestsdk, nlohmann/json.
4.2.1 Функціональні можливості IoT-клієнта
1. Збір та передача сенсорних даних
   1. Збір значень показників навколишнього середовища.
2. Передача даних на сервер
   1. Використання JSON-формату для передачі даних.
4.2.2 Конфігурація IoT-клієнта
        Система конфігурується за допомогою JSON файлу, що програма зчитує перед початком роботи. Ключовими параметрами є: адреса API для обробки даних, дані про сенсор, інтервал для збору показників (у секундах).
