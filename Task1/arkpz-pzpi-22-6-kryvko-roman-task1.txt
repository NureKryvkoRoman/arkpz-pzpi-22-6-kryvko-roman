1. Business Requirements
   1.1. Background
У сучасному світі агропромисловий сектор відіграє ключову роль у забезпеченні продовольчої безпеки, а тепличне господарство є важливим елементом цього сектора. Зі збільшенням попиту на екологічно чисту продукцію та необхідністю знижувати витрати на вирощування сільськогосподарських культур дедалі більше фермерів звертаються до автоматизації теплиць. Водночас ручний моніторинг і управління мікрокліматом у теплицях є трудомісткими процесами, які вимагають значних людських ресурсів і не завжди забезпечують точність та своєчасність рішень.
Програмна система для автоматичного контролю стану рослин у теплицях створюється для розв’язання цих проблем, забезпечуючи ефективний моніторинг ключових параметрів, таких як температура, вологість та освітленість, а також автоматизацію управління поливом і штучним освітленням. Система допоможе знизити ризики втрати врожаю через несприятливі умови, зменшити витрати на ручне обслуговування теплиць і підвищити продуктивність.
Крім цього, інтеграція IoT-сенсорів із серверною частиною забезпечить аналіз даних у реальному часі та формування історичних звітів, що дозволить фермерам краще розуміти потреби рослин і адаптувати підходи до їх вирощування. Веб-інтерфейс та мобільний додаток у майбутньому стануть інструментами для зручного дистанційного керування теплицями, надаючи можливість оперативно реагувати на зміни.
Ця система є відповіддю на зростаючий попит на сучасні автоматизовані рішення в аграрному секторі, забезпечуючи фермерам конкурентну перевагу та сприяючи розвитку тепличного господарства.
   1.2. Business Opportunity
    Фермери прагнуть оптимізувати витрати ресурсів і підвищити врожайність. Наявні рішення часто або занадто дорогі для невеликих тепличних господарств, або не забезпечують необхідної гнучкості для різних умов. Розробка системи для автоматичного контролю стану рослин у теплицях відкриває можливості для:
    - Оптимізації процесів вирощування через автоматизацію;
    - Скорочення витрат на воду, електроенергію та людські ресурси;
    - Підвищення врожайності за рахунок точного моніторингу параметрів;
    - Уникнення втрат через виявлення критичних ситуацій у реальному часі.

2. Vision of the Solution
    2.1. Vision Statement
    Система автоматичного контролю стану рослин у теплицях змінить підхід до управління тепличним господарством, роблячи його більш ефективним, економічним та екологічним. Місія системи — створити екосистему, яка зробить тепличне господарство доступним навіть для новачків, одночасно надаючи професійним фермерам інструменти для більш точного управління. Таким чином, система стане універсальним рішенням для фермерів, які прагнуть підвищити врожайність, мінімізувати витрати та покращити екологічну стійкість своєї діяльності. Система забезпечить ефективне використання ресурсів і значно полегшить догляд за рослинами завдяки інтеграції сучасних IoT-рішень, мобільних технологій і веб-інтерфейсів. Користувачі зможуть у режимі реального часу передивлятися умови вирощування, отримувати сповіщення про критичні зміни та керувати пристроями дистанційно. Ця система сприятиме підвищенню врожайності, зменшенню витрат і мінімізації втручання людини в рутинні процеси, роблячи теплиці більш автономними та екологічно ефективними.
    Одним із головних конкурентів для системи автоматичного контролю стану рослин у теплицях є "Growlink". Це IoT-платформа для моніторингу та автоматизації теплиць, яка пропонує широкий набір функцій, включаючи сенсори для моніторингу кліматичних умов, автоматичне управління пристроями, а також мобільний додаток і веб-інтерфейс. Проте "Growlink" має кілька недоліків, які відкривають можливості для нашої системи:
    - Висока вартість: Система орієнтована на великих клієнтів із значними фінансовими ресурсами, що робить її недоступною для дрібних фермерів.
    - Обмежена локалізація: Сервіс зосереджений на американському та європейському ринках, без повної адаптації до локальних умов інших регіонів.
    - Закрита екосистема: Відсутність відкритого API або гнучких можливостей для кастомізації обмежує інтеграцію з іншими рішеннями.
    Основні переваги нашої системи:
    - Доступності для малих і середніх фермерів завдяки помірній вартості.
    - Можливості підтримки локальних мов, включаючи українську.
    - Зручний інтерфейс, а також інтуїтивно зрозумілий дизайн для надання кращого користувацького досвіду
    Відкрите API для сторонніх розробників.
    - Можливості монетизації:
    - Підписка на хмарний сервіс: Регулярна оплата за доступ до хмарного зберігання даних, звітів і аналітики.
    - Преміум-функції: Наприклад, інтелектуальні рекомендації, доступні лише для користувачів, що сплачують за них.

   2.2. Major Features
    IoT сенсори і контроль:
    F1: Вимірювання температури, вологості та рівня освітлення у реальному часі.
    F2: Автоматичне або ручне керування поливом і освітленням через мобільний додаток або веб-інтерфейс.
    Мобільний додаток:
    F3: Перегляд поточних показників і історії даних.
    F4: Сповіщення про аномальні зміни або несправності.
    F5: Інтуїтивно зрозумілий інтерфейс для дистанційного управління.
    Веб-інтерфейс:
    F6: Можливість аналізу історії показників і формування звітів.
    F7: Інструменти для ручного налаштування параметрів.
    F8: Система оновлення та візуалізації даних в реальному часі.
    Серверна частина:
    F9: API для передачі даних від сенсорів до мобільного додатка та веб-інтерфейсу.
    F10: Зберігання історії отриманих даних у базі даних.
    F11: Інтелектуальні алгоритми для виявлення критичних ситуацій.

3. Scope and Limitations
   3.1. Scope of Initial Release
    На поточному етапі розробки система автоматичного контролю стану рослин у теплицях буде зосереджена на реалізації основного функціоналу серверної частини та IoT-компонентів.
      3.1.1. IoT-компоненти
    - Інтеграція сенсорів для моніторингу температури, вологості та рівня освітлення.
    - Розробка механізмів автоматичного керування пристроями, такими як системи поливу та освітлення.
      3.1.2. Серверна частина
    - Реалізація серверної частини для прийому, обробки та зберігання даних, отриманих від IoT-сенсорів.
    - API для взаємодії з клієнтськими додатками та надсилання команд контролерам у теплиці.
    - Забезпечення зберігання історичних даних для подальшого аналізу та використання.
    - Алгоритми для автоматичного визначення аномалій у даних і сповіщення користувача про необхідність втручання.
   3.2. Scope of Subsequent Releases
      3.2.1. Веб-інтерфейс
    - Візуалізація даних сенсорів у реальному часі.
    - Надання доступу до історії показників, звітів та налаштувань автоматизації.
    - Інтуїтивний інтерфейс для ручного керування параметрами теплиці.
      3.2.2. Мобільний додаток
    - Оперативний перегляд стану рослин і отримання сповіщень про аномалії.
    - Функціонал для віддаленого керування теплицею через смартфон.
    - Історія даних та рекомендації на основі аналізу показників.

4. Business Context
    4.1. Operating Environment
    Система для автоматичного контролю стану рослин у теплицях буде складатися із серверної частини, IoT-компонентів та веб-застосунку. Серверна частина буде реалізована за допомогою фреймворку Spring на мові програмування Java, забезпечуючи обробку, зберігання даних і надання API для клієнтських застосунків. Для збереження даних використовуватиметься реляційна база даних MariaDB. Веб-застосунок буде розроблений з використанням фреймворку React, забезпечуючи зручний інтерфейс для перегляду стану теплиці, історії даних і керування пристроями. IoT-компоненти будуть інтегровані з сенсорами, які вимірюють температуру, вологість і освітленість, а також пристроями для автоматичного керування системами поливу та освітлення.
